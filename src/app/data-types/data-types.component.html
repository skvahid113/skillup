Numeric Data Types

  • INT (Integer): Syntax: INT

    Description: The INT data type is used for storing whole numbers without fractional components. It is commonly used for representing quantities, counts, or unique identifiers.

    SQL Example: 
    
CREATE TABLE Employee (
  EmployeeID INT,
  Salary INT
);
  • DECIMAL (Decimal): Syntax: DECIMAL(precision, scale)

    Description: The DECIMAL data type is used for representing fixed-point decimal numbers with a specified precision and scale. It is ideal for financial and monetary values where precision is crucial.

    SQL Example: 
    
CREATE TABLE Product (
  ProductID INT,
  Price DECIMAL(10, 2)
);
  • FLOAT (Floating-Point): Syntax: FLOAT(precision)

    Description: The FLOAT data type is used to store approximate numeric values with fractional components. It is commonly used in scientific and engineering calculations.

    SQL Example: 
    
CREATE TABLE SensorData (
  ReadingID INT,
  Value FLOAT(2)
);
  • NUMERIC (Numeric): Syntax: NUMERIC(precision, scale)

    Description: The NUMERIC data type is similar to DECIMAL, used for representing fixed-point decimal numbers with a specified precision and scale. It is suitable for exact numerical values requiring high precision.

    SQL Example: 
    
CREATE TABLE FinancialData (
  TransactionID INT,
  Amount NUMERIC(12, 4)
);
  • SMALLINT (Small Integer): Syntax: SMALLINT

    Description: The SMALLINT data type is used to store small whole numbers, often used for quantities, counts, or attributes with a limited range.

    SQL Example: 
    
CREATE TABLE Room (
  RoomID INT,
  Capacity SMALLINT
);
  • BIGINT (Big Integer): Syntax: BIGINT

    Description: The BIGINT data type is used to store very large whole numbers, making it suitable for counting large populations or generating unique identifiers.

    SQL Example: 
    
CREATE TABLE SocialMedia (
  UserID BIGINT,
  FollowersCount BIGINT
);

Character String Data Types

  • CHAR (Fixed-length character): Syntax: CHAR(length)

    Description: The CHAR data type stores fixed-length strings with a specified length. It is commonly used for representing codes or identifiers with a consistent format.

    SQL Example: 
    
CREATE TABLE Product (
  ProductID INT,
  SKU CHAR(10)
);
  • VARCHAR (Variable-length character): Syntax: VARCHAR(length)

    Description: The VARCHAR data type stores variable-length character strings. It is ideal for storing textual data with varying lengths, such as names, descriptions, or addresses.

    SQL Example: 
    
CREATE TABLE Customer (
  CustomerID INT,
  Name VARCHAR(50)
);
  • TEXT (Long text): Syntax: TEXT

    Description: The TEXT data type is used for storing long text or document-like content, including articles, reviews, or other textual data with no predefined length limit.

    SQL Example: 
    
CREATE TABLE Article (
  ArticleID INT,
  Content TEXT
);
  • NVARCHAR (Unicode variable-length character): Syntax: NVARCHAR(length)

    Description: The NVARCHAR data type is similar to VARCHAR but stores Unicode character strings. It's suitable for multilingual or international text data.

    SQL Example: 
    
CREATE TABLE MultilingualData (
  EntryID INT,
  Text NVARCHAR(100)
);
  • CLOB (Character Large Object): Syntax: CLOB

    Description: The CLOB data type is designed for storing extremely long text or documents, such as novels, research papers, or large textual content.

    SQL Example: 
    
CREATE TABLE Novel (
  NovelID INT,
  Content CLOB
);
  • STRING (General string type): Syntax: STRING

    Description: The STRING data type represents a general-purpose string with varying length, commonly used for custom data formats, user-generated content, or flexible text fields.

    SQL Example: 
    
CREATE TABLE CustomData (
  DataID INT,
  Text STRING
);

Date and Time Data Types

  • DATE (Date): Syntax: DATE

    Description: The DATE data type is used for storing calendar dates. It is commonly used for recording birthdates, event dates, and other date-related information.

    SQL Example: 
    
CREATE TABLE Event (
  EventID INT,
  EventDate DATE
);
  • TIME (Time): Syntax: TIME

    Description: The TIME data type represents a point in time within a day, useful for recording time values such as meeting times, flight departure times, or opening hours.

    SQL Example: 
    
CREATE TABLE Meeting (
  MeetingID INT,
  StartTime TIME
);
  • DATETIME (Date and Time): Syntax: DATETIME

    Description: The DATETIME data type stores both date and time components, making it suitable for recording event timestamps, sensor data timestamps, and other time-related information.

    SQL Example: 
    
CREATE TABLE LogEntry (
  LogEntryID INT,
  LogTimestamp DATETIME
);
  • TIMESTAMP (Timestamp with Time Zone): Syntax: TIMESTAMP WITH TIME ZONE

    Description: The TIMESTAMP data type includes a time zone information, making it suitable for global event scheduling, flight schedules, and international conference call timings.

    SQL Example: 
    
CREATE TABLE FlightSchedule (
  FlightID INT,
  DepartureTime TIMESTAMP WITH TIME ZONE
);
  • INTERVAL (Time Interval): Syntax: INTERVAL

    Description: The INTERVAL data type represents a duration between two timestamps, ideal for countdown timers, time-based notifications, and event duration calculations.

    SQL Example: 
    
CREATE TABLE Countdown (
  CountdownID INT,
  TimeDuration INTERVAL
);
  • TIMETZ (Time with Time Zone): Syntax: TIMETZ

    Description: The TIMETZ data type combines time values with time zone information, suitable for handling meeting schedules in different time zones, global broadcast schedules, and more.

    SQL Example: 
    
CREATE TABLE MeetingSchedule (
  MeetingID INT,
  MeetingTime TIMETZ
);

Binary Data Types

  • BLOB (Binary Large Object): Syntax: BLOB

    Description: The BLOB data type is used for storing large binary data, such as images, audio, or video files. It's commonly used for document attachments, encryption keys, or raw sensor data.

    SQL Example: 
    
CREATE TABLE Images (
  ImageID INT,
  ImageData BLOB
);
  • BYTEA (Binary Data): Syntax: BYTEA

    Description: The BYTEA data type is used for storing binary data, including image avatars, biometric data, digital signatures, and binary code or executables.

    SQL Example: 
    
CREATE TABLE Users (
  UserID INT,
  ProfileImage BYTEA
);
  • VARBINARY (Variable-length binary): Syntax: VARBINARY(length)

    Description: The VARBINARY data type stores variable-length binary data, ideal for image thumbnails, audio snippets, and custom binary formats.

    SQL Example: 
    
CREATE TABLE CustomData (
  DataID INT,
  BinaryContent VARBINARY(100)
);
  • BIT (Bit Strings): Syntax: BIT(length)

    Description: The BIT data type is used for storing binary flags, permissions, or boolean values in a compact format.

    SQL Example: 
    
CREATE TABLE Permissions (
  PermissionID INT,
  AccessFlags BIT(8)
);
  • RAW (Raw Binary Data): Syntax: RAW(length)

    Description: The RAW data type is designed for low-level binary data storage, machine code, or physical sensor data.

    SQL Example: 
    
CREATE TABLE MachineCode (
  CodeID INT,
  BinaryCode RAW(64)
);
  • IMAGE (Image Data): Syntax: IMAGE

    Description: The IMAGE data type is used for storing high-resolution images, medical image data, artwork, geospatial images, or custom image formats.

    SQL Example: 
    
CREATE TABLE Artwork (
  ArtworkID INT,
  ArtImage IMAGE
);
