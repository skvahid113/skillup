Inner Join

An Inner Join returns only the matching rows from both tables involved in the join. It retrieves rows where there is a match between the specified columns in both tables.

Create Orders Table Query

CREATE TABLE Orders ( OrderID INT PRIMARY KEY, CustomerID INT, FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID) );

Full Outer Join

A Full Outer Join returns all rows when there is a match in either the left or the right table. If there's no match, NULL values are returned for the non-matching table.

Create Customers Table Query

CREATE TABLE Customers ( CustomerID INT PRIMARY KEY, CustomerName VARCHAR(255) );

Create Orders Table Query

CREATE TABLE Orders ( OrderID INT PRIMARY KEY, CustomerID INT, FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID) );

Full Outer Join Query

Explanation: In this Full Outer Join example, we retrieve all customer data and their matching orders, or all order data and their matching customers. If there's no match, NULL values are shown for the non-matching columns. It includes columns from both "Customers" and "Orders" tables in the output.

Cross Join

A Cross Join (Cartesian Join) returns the Cartesian product of two tables, which means it combines every row from the first table with every row from the second table.

Create Customers Table Query

CREATE TABLE Customers ( CustomerID INT PRIMARY KEY, CustomerName VARCHAR(255) );

Create Orders Table Query

CREATE TABLE Orders ( OrderID INT PRIMARY KEY, CustomerID INT, FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID) );
