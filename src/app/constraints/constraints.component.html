NOT NULL Constraint

The NOT NULL constraint enforces that a column cannot contain NULL values, meaning that every row must have a non-null value in that column.

Syntax: column_name data_type NOT NULL

  • Ensuring that the 'username' column in a 'User' table cannot be NULL:
    • 
CREATE TABLE User (
  UserID INT PRIMARY KEY,
  Username VARCHAR(50) NOT NULL,
  -- Other columns...
);
  • Requiring the 'email' column in a 'Customer' table to be non-null:
    • 
CREATE TABLE Customer (
  CustomerID INT PRIMARY KEY,
  Email VARCHAR(100) NOT NULL,
  -- Other columns...
);
  • Forcing the 'product_name' column in a 'Product' table to have a non-null value:
    • 
CREATE TABLE Product (
  ProductID INT PRIMARY KEY,
  ProductName VARCHAR(255) NOT NULL,
  -- Other columns...
);

DEFAULT Constraint

The DEFAULT constraint allows you to specify a default value for a column when no value is explicitly provided during an INSERT operation.

Syntax: column_name data_type DEFAULT default_value

  • Setting a default value of 'active' for the 'status' column in a 'User' table:
    • 
CREATE TABLE User (
  UserID INT PRIMARY KEY,
  Username VARCHAR(50),
  Status VARCHAR(20) DEFAULT 'active',
  -- Other columns...
);
  • Defining a default value of 'N/A' for the 'middle_name' column in an 'Employee' table:
    • 
CREATE TABLE Employee (
  EmployeeID INT PRIMARY KEY,
  FirstName VARCHAR(50),
  MiddleName VARCHAR(50) DEFAULT 'N/A',
  -- Other columns...
);
  • Using a default value of 0 for the 'points' column in a 'Player' table:
    • 
CREATE TABLE Player (
  PlayerID INT PRIMARY KEY,
  PlayerName VARCHAR(50),
  Points INT DEFAULT 0,
  -- Other columns...
);

CHECK Constraint

The CHECK constraint allows you to specify a condition that values in a column must satisfy. It ensures that data entered into the column meets specific criteria.

Syntax: column_name data_type CHECK (condition)

  • Enforcing that the 'age' column in a 'Person' table must be greater than or equal to 18:
    • 
    CREATE TABLE Person (
      PersonID INT PRIMARY KEY,
      Name VARCHAR(50),
      Age INT CHECK (Age >= 18),
      -- Other columns...
    );
  • Checking that the 'total_price' column in an 'Order' table is greater than zero:
    • 
    CREATE TABLE Order (
      OrderID INT PRIMARY KEY,
      ProductName VARCHAR(100),
      TotalPrice DECIMAL(10, 2) CHECK (TotalPrice > 0),
      -- Other columns...
    );
  • Using a CHECK constraint to ensure that the 'rating' column in a 'Review' table falls within a valid range:
    • 
    CREATE TABLE Review (
      ReviewID INT PRIMARY KEY,
      Title VARCHAR(255),
      Rating INT CHECK (Rating >= 1 AND Rating <= 5),
      -- Other columns...
    );

UNIQUE Constraint

The UNIQUE constraint ensures that all values in a column are unique across rows in a table. It is used to prevent duplicate entries in the specified column.

Syntax: UNIQUE (column_name)

  • Applying a UNIQUE constraint to the 'email' column in a 'User' table to enforce email uniqueness: 
    • 
    CREATE TABLE User (
      UserID INT PRIMARY KEY,
      Email VARCHAR(100) UNIQUE,
      -- Other columns...
    );
  • Ensuring that the 'serial_number' column in a 'Product' table contains unique values:
    • 
    CREATE TABLE Product (
      ProductID INT PRIMARY KEY,
      SerialNumber VARCHAR(50) UNIQUE,
      -- Other columns...
    );
  • Using a UNIQUE constraint to enforce uniqueness in the 'ISBN' column of a 'Book' table:
    • 
    CREATE TABLE Book (
      BookID INT PRIMARY KEY,
      Title VARCHAR(255),
      ISBN VARCHAR(20) UNIQUE,
      -- Other columns...
    );

PRIMARY KEY Constraint

The PRIMARY KEY constraint uniquely identifies each row in a table. It enforces the uniqueness and non-null property of a column, typically used as the main identifier for a table.

Syntax: column_name data_type PRIMARY KEY

  • Defining the 'employee_id' column as the primary key in an 'Employee' table:
    • 
    CREATE TABLE Employee (
      EmployeeID INT PRIMARY KEY,
      FirstName VARCHAR(50),
      LastName VARCHAR(50),
      -- Other columns...
    );
  • Using 'product_code' as the primary key in a 'Product' table:
    • 
    CREATE TABLE Product (
      ProductCode VARCHAR(10) PRIMARY KEY,
      ProductName VARCHAR(255),
      -- Other columns...
    );
  • Applying the primary key constraint to the 'student_id' column in a 'Student' table:
    • 
    CREATE TABLE Student (
      StudentID INT PRIMARY KEY,
      Name VARCHAR(100),
      -- Other columns...
    );

FOREIGN KEY Constraint

The FOREIGN KEY constraint establishes a link between two tables by specifying that values in a column must match the values in another table's primary key. It enforces referential integrity.

Syntax: FOREIGN KEY (column_name) REFERENCES referenced_table (referenced_column)

  • Creating a foreign key constraint in the 'Order' table referencing the 'Customer' table:
    • 
    CREATE TABLE Order (
      OrderID INT PRIMARY KEY,
      CustomerID INT,
      OrderDate DATE,
      FOREIGN KEY (CustomerID) REFERENCES Customer (CustomerID),
      -- Other columns...
    );
  • Enforcing a foreign key constraint in the 'Review' table referencing the 'Product' table:
    • 
    CREATE TABLE Review (
      ReviewID INT PRIMARY KEY,
      ProductID INT,
      Rating INT,
      FOREIGN KEY (ProductID) REFERENCES Product (ProductID),
      -- Other columns...
    );
  • Defining a foreign key constraint in the 'Employee' table referencing the 'Department' table:
    • 
    CREATE TABLE Employee (
      EmployeeID INT PRIMARY KEY,
      Name VARCHAR(100),
      DepartmentID INT,
      FOREIGN KEY (DepartmentID) REFERENCES Department (DepartmentID),
      -- Other columns...
    );
